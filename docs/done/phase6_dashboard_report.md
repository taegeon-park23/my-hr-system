# Phase 6: 프론트엔드 대시보드 확장 보고서

## 1. 작업 개요
- **작업명:** 관리자용 대시보드 위젯 추가 및 실데이터 연동
- **완료일:** 2025-12-18
- **핵심 성과:** 하드코딩된 데이터를 제거하고 백엔드 통계 API와 연동하여 조직 현황을 한눈에 파악 가능하도록 개선.

## 2. 상세 작업 내역

### 2.1 대시보드 통계 API 연동
- **`useDashboardSummary` 훅 생성:** 백엔드의 `/api/dashboard/summary` 엔드포인트와 연동하여 전사적 통계(총원, 승인 대기, 오늘 출근 등)를 가져오는 기능 구현.
- **`useDepartmentStats` 훅 생성:** `/api/dashboard/dept-stats` 엔드포인트 연동으로 부서별 인원 현황 데이터 확보.

### 2.2 관리자 전용 위젯 구현
- **ManagerStats 위젯:**
  - 조직 전체의 핵심 지표(전체 임직원, 오늘 출근자, 결재 대기 건수, 반려 건수)를 카드 형태로 시각화.
  - 권한별 분기 처리를 통해 `USER` 권한을 제외한 관리자 계층(`DEPT_MANAGER` 이상)에게만 노출.
- **DepartmentHeadcount 위젯:**
  - 부서별 인원 분포를 수평 바 차트(Pure CSS/Tailwind 활용)로 구현.
  - 별도의 차트 라이브러리 없이도 직관적으로 부서별 규모를 비교 가능하도록 시각화.

### 2.3 메인 대시보드 레이아웃 개선
- 기존 개인용 `QuickStats` 아래에 관리자용 정보를 배치하는 그리드 레이아웃 적용.
- 조직 정보가 강조되도록 디자인 톤앤매너 조정.

## 3. 검증 결과
- 관리자 계정으로 접속 시 실제 DB에 저장된 부서 및 인원 정보가 대시보드에 정확하게 집계되어 표시됨을 확인.
- 일반 사용자 계정으로 접속 시 조직 통계 위젯이 노출되지 않음을 확인.
