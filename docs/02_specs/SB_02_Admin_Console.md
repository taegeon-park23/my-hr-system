# StoryBoard: 관리자 콘솔 (SB-02)

## 0. 개요
본 문서는 기업 관리자(Tenant Admin) 및 HR 담당자가 조직을 관리하고 운영 현황을 모니터링하는 **Admin Console**의 주요 화면을 정의한다.

---

## **1. SCR-ADM-01: HR 현황 대시보드 (Admin Dashboard)**

### **1.1 개요**
*   **목적**: 전사 인원, 휴가 사용률, 근태 이상 징후 등 핵심지표(KPI)를 모니터링한다.
*   **사용자**: Tenant Admin, HR Manager
*   **연결 Spec**: `SPEC_11`

### **1.2 레이아웃**
*   **Key Metrics Row**:
    1.  **총 인원**: 전사 인원 수 (전월 대비 증감 표시).
    2.  **휴가 사용률**: 전사 평균 연차 소진율 (Gauge Chart).
    3.  **금일 휴가자**: 오늘 부재중인 인원 수.
    4.  **근태 이상**: 지각/결근 발생 건수 (금일).
*   **Chart Section**:
    *   **부서별 휴가 사용 현황**: Bar Chart (부서별 비교).
    *   **월별 입/퇴사 추이**: Line Chart.
*   **Widgets**:
    *   **공지사항 관리**: 최근 공지 리스트 및 "글쓰기" 버튼.
    *   **Todo**: 결재 처리 대기 중인 문서 중 장기 계류 건 알림.

---

## **2. SCR-ADM-02: 구성원 관리 (User Management)**

### **2.1 개요**
*   **목적**: 직원 정보를 등록, 수정하고 계정을 관리(활성/비활성, 비밀번호 초기화)한다.
*   **연결 Spec**: `SPEC_04`

### **2.2 레이아웃**
*   **Toolbar**:
    *   **Search**: 이름, 부서, 이메일 검색.
    *   **Filter**: 재직 상태(재직/휴직/퇴사), 부서 필터.
    *   **Action**: [직원 등록(단건)], [엑셀 일괄 등록], [내보내기].
*   **Data Grid**:
    *   Cols: 이름(프로필사진) | 사번 | 부서 | 직책 | 입사일 | 상태 | Action(수정)
    *   **Badge System**: 상태값은 Color Badge로 구분 (`Active`=Green, `Resigned`=Gray).

### **2.3 인터랙션**
*   **직원 등록 모달**:
    *   **Tab 1 (기본정보)**: 이름, 이메일, 사번, 입사일.
    *   **Tab 2 (조직정보)**: 부서(Tree Selector), 직위/직책.
    *   **Tab 3 (계정정보)**: 권한(Role), 초기 비밀번호 설정.

---

## **3. SCR-ADM-03: 조직도 관리 (Organization Tree)**

### **3.1 개요**
*   **목적**: 회사의 부서 구조를 시각적으로 편집하고 관리한다.
*   **연결 Spec**: `SPEC_04`

### **3.2 레이아웃**
*   **Split Layout (Left: Tree / Right: Detail)**
*   **Left (Tree View)**:
    *   계층형 트리 구조 (Root Company -> Divison -> Team).
    *   각 노드 우클릭 메뉴: [하위 부서 추가], [이름 변경], [삭제], [이동].
    *   Drag & Drop 지원 (부서 이동).
*   **Right (Detail Panel)**:
    *   선택된 부서의 상세 정보 (부서장 지정, 부서 코드).
    *   **소속원 리스트**: 해당 부서에 소속된 직원 목록 표시.

### **3.3 인터랙션**
*   **부서장 지정**: 소속원 리스트에서 "부서장(Manager)" 체크박스 활성화 시 해당 부서의 리더로 지정되고, 결재 라인의 승인권자가 됨.

---

## **4. SCR-ADM-04: 결재 문서 전체 조회 (All Documents)**

### **4.1 개요**
*   **목적**: HR 관리자가 사내의 모든 결재 문서를 모니터링하고, 필요 시 전결 처리(Admin Action)를 수행한다.
*   **연결 Spec**: `SPEC_01`

### **4.2 레이아웃**
*   **Advanced Filter**:
    *   기간 설정, 휴가 유형별, 부서별, 기안자별 복합 검색 지원.
*   **Status Tabs**: 전체 | 진행중 | 완료 | 반려
*   **Admin Action Column**:
    *   일반 사용자는 볼 수 없는 "관리자 옵션" 버튼 (`...`) 노출.

### **4.3 인터랙션**
*   **강제 승인/반려 (Force Action)**:
    *   결재자가 장기 부재중이거나 시스템 오류 시, 관리자가 강제로 상태를 변경할 수 있는 기능.
    *   클릭 시 **사유 입력 모달** 필수 (Audit Log에 기록됨: `SPEC_07`).
